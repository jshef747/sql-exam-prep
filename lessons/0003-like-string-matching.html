like — חיפוש לפי תבנית טקסט

איך מסננים לפי "מכיל", "מתחיל ב‑", "מסתיים ב‑". שאלה שחוזרת כמעט בכל בחינה: "מצא לקוחות ששם משפחתם מכיל את הרצף 'tov'".

🎯 היעד: לכתוב תנאי where שמתאים מחרוזות חלקיות בעזרת like והתווים % ו‑_.
↩️ עד כה: שיעור 1 — select/from/where; שיעור 2 — צירוף טבלאות. עכשיו נחדד את ה‑where: סינון לפי תבנית ולא רק לפי שוויון מדויק.

הסכמה שנעבוד עליה

שוב סכמת חברת ביטוח הרכב. נתמקד בעמודות טקסט: fname, lname, city, model, brand.

טבלהעמודות (רלוונטיות)
personid, fname, lname, street, city, birth
ownsid, license
carlicense, model, brand, year

סכמה מלאה ב‑מילון המונחים והסכמות ←

1האופרטור like ושני תווי הבר

במקום = (שוויון מדויק) משתמשים ב‑like כדי להתאים תבנית. שני תווים מיוחדים (מצגת פרק 3, שקופית 17):

תומשמעות
%כל רצף תווים — כולל רצף ריק (אפס תווים או יותר).
_תו בודד אחד בדיוק.

למצוא את כל מי ששם משפחתו מכיל את הרצף 'tov' (איפשהו בתוך השם):

select fname, lname
from   person
where  lname like '%tov%'

ה‑% משני הצדדים אומר: "כל דבר, אחר כך tov, אחר כך כל דבר" — כלומר tov בכל מקום במחרוזת.

2ספר התבניות

כל התבניות הנפוצות במקום אחד. שימו לב איפה ה‑% יושב:

תבניתמתאימה ל…דוגמה שמתאימה
'tov%'מתחיל ב‑tovtov, tover
'%tov'מסתיים ב‑tovshemtov, simantov
'%tov%'מכיל tov איפשהוshem tov levi
't_v't, תו בודד כלשהו, ואז v — בדיוק 3 תוויםtov, tav (לא toov)
'a%n'מתחיל ב‑a ומסתיים ב‑naaron, an
💡 _ = בדיוק תו אחד; % = אפס תווים או יותר. 't_v' דורש אורך 3 בדיוק; 't%v' מקבל גם tv וגם toooov.

3אותיות גדולות/קטנות (case sensitive)

במצגת הקורס (פרק 3, שקופית 17) נכתב במפורש: התאמת תבניות היא case sensitive. כלומר '%tov%' לא יתאים ל‑'Tov' או ל‑'TOV'.

⚠️ בבחינה שמופיע בה רצף כמו 'tov', בדרך כלל נאמר "הניחו שכל השמות שמורים באותיות קטנות". סמכו על ההנחה הזו וכתבו את התבנית באותיות קטנות. אם לא נאמר — שווה לציין את ההנחה בתשובה.

4שילוב like עם צירוף — שאלת בחינה אמיתית

🎓 שאלה אמיתית — מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480, קב' 13111202) · שאלה 6.2 · סכמה ב' — חברת ביטוח רכב:
"מצא את התאונות שמעורבים בהן רכבים של לקוחות ששם משפחתם מכיל 'tov'. הצג מספר תאונה, מספר רכב, תאריך, ושם מלא של הבעלים."

השם נמצא ב‑person, הקשר לרכב ב‑owns, והתאונה ב‑accident — צירוף של שלוש טבלאות, ועליו תנאי like:

select distinct accident.accident_id, accident.license, accident.date,
                person.fname, person.lname
from   accident join owns   on accident.license = owns.license
               join person on owns.id = person.id
where  person.lname like '%tov%'

— אותו דבר עם פסיק ו‑where:

select distinct accident.accident_id, accident.license, accident.date,
                person.fname, person.lname
from   accident, owns, person
where  accident.license = owns.license
  and  owns.id = person.id
  and  person.lname like '%tov%'

הכול מתחבר: שיעור 2 (צירוף שלוש טבלאות) + שיעור 3 (like). זה בדיוק סוג השאלה שתפגשו בחלק ב'. (ה‑distinct כאן הוא ליתר ביטחון בלבד — בשאילתה הזו הוא מיותר; ההסבר המלא בתרגיל 4.)

תרגול — כתבו ובדקו את עצמכם ✍️

נסו בעצמכם לפני "הצג פתרון". (אפשר לבחור מצב עריכה — השלמה / צביעה / רגיל — בפינה השמאלית העליונה.)

תרגיל 1 — הצג שם פרטי ושם משפחה של כל מי ששם משפחתו מכיל את הרצף 'tov'.
select fname, lname
from   person
where  lname like '%tov%'

"מכיל" ⟵ % משני הצדדים.

תרגיל 2 — הצג את האנשים ששמם הפרטי מתחיל באות 'A'.
select fname, lname
from   person
where  fname like 'A%'

"מתחיל ב‑" ⟵ ה‑% רק בסוף. (זכרו: case sensitive — 'A%' לא יתאים ל‑'a...'.)

תרגיל 3 — הצג את license ו‑model של כל הרכבים שהדגם שלהם מכיל את הרצף 'X'.
select license, model
from   car
where  model like '%X%'

אותו דפוס "מכיל", הפעם על עמודת model.

🎓 תרגיל 4 (שאלה ממבחן) — הצג accident_id, license, date ושם מלא של הבעלים, עבור תאונות של רכבים שבעליהם שם משפחתם מכיל 'tov'.

📄 מקור: מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480) · שאלה 6.2 · סכמה ב' — חברת ביטוח רכב

select distinct accident.accident_id, accident.license, accident.date,
                person.fname, person.lname
from   accident join owns   on accident.license = owns.license
               join person on owns.id = person.id
where  person.lname like '%tov%'

— אותו דבר עם פסיק ו‑where:

select distinct accident.accident_id, accident.license, accident.date,
                person.fname, person.lname
from   accident, owns, person
where  accident.license = owns.license
  and  owns.id = person.id
  and  person.lname like '%tov%'

צירוף שלוש טבלאות (accident → owns → person) + תנאי like. זו בדיוק שאלה 6.2 מהבחינה.
למה distinct? בשאילתה הזו הוא למעשה מיותר: שם הבעלים (fname, lname) נמצא ב‑select, וכל צירוף של (תאונה ↔ בעלים) הוא ייחודי — אז אין שתי שורות זהות שצריך לאחד. רכב עם שני בעלים שעונים על התנאי ייתן שתי שורות שונות (שם שונה), ו‑distinct ממילא לא מאחד אותן. הוא היה נחוץ רק אילו היינו משמיטים את שם הבעלים מהפלט (למשל רק accident_id, license, date) — אז לרכב עם כמה בעלים תואמים היו צצות שורות זהות. נהוג להוסיף distinct ליתר ביטחון בצירופים, אבל כדאי לדעת מתי הוא באמת עובד.

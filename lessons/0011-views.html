תצפיות · create view

תצפית (view) היא שאילתה שמורה בשם שמתנהגת כמו טבלה. כותבים פעם אחת שאילתה מורכבת, נותנים לה שם — ומאז שולפים ממנה כאילו הייתה טבלה רגילה. הבחינה לפעמים מבקשת במפורש "הגדר תצפית".

🎯 היעד: להגדיר תצפית עם create view ... as, לשלוף ממנה כמו מטבלה, ולבנות עליה שאילתות נוספות (סינון, מיון, הקבצה).
↩️ עד כה למדנו לכתוב שאילתות (שיעורים 1–10). תצפית פשוט נותנת לשאילתה שם ושומרת אותה לשימוש חוזר. אין כאן תחביר שליפה חדש — רק עטיפה.

1מה זו תצפית?

תצפית = שאילתה ששמרנו תחת שם. נקודות מפתח (מצגת תיאור שפת SQL, נושא ה‑view):

  • היא טבלה וירטואלית — שולפים ממנה בדיוק כמו מטבלה: select ... from <שם התצפית>.
  • היא לא שומרת נתונים פיזית. בכל פנייה השאילתה שמאחוריה מחושבת מחדש — לכן היא תמיד מעודכנת לנתונים הנוכחיים.
  • שימושית כדי לא לחזור שוב ושוב על אותה שאילתה מורכבת, ולתת לה שם בעל משמעות.

2התחביר — create view ... as

מגדירים תצפית פעם אחת, ואז שולפים ממנה. הריצו (▶) את שני המשפטים ביחד:

create view cars_per_brand as
  select brand, count(*) as num_cars
  from   car
  group by brand;

select * from cars_per_brand

אחרי ה‑create view, cars_per_brand הוא שם שאפשר לשלוף ממנו כמו מכל טבלה — והוא מחזיר שורה לכל יצרן עם מספר רכביו.

💡 שמות העמודות של התצפית נקבעים מה‑select שלה. כאן נתנו במפורש שם לעמודה המחושבת באמצעות as num_cars — זו הרגלה טובה, כי אחרת לעמודה יהיה שם ברירת‑מחדל מסורבל כמו COUNT(*).

3בונים שאילתה מעל התצפית

הכוח האמיתי: אחרי שיש תצפית, אפשר לסנן / למיין / להקביץ מעליה כאילו הייתה טבלה. למשל — רק יצרנים עם 3 רכבים או יותר, ממוין יורד:

create view cars_per_brand as
  select brand, count(*) as num_cars
  from   car
  group by brand;

select * from cars_per_brand
where  num_cars >= 3
order by num_cars desc

וגם הקבצה מעל התצפית — מה המספר הגבוה ביותר של רכבים ליצרן:

create view cars_per_brand as
  select brand, count(*) as num_cars
  from   car
  group by brand;

select max(num_cars) as most
from   cars_per_brand
📌 שימו לב כמה זה קריא: select max(num_cars) from cars_per_brand במקום לקנן את כל ה‑group by בתוך תת‑שאילתה. זו בדיוק מטרת התצפית — להסתיר מורכבות מאחורי שם.

4תצפית מעל צירוף והקבצה

תצפיות זורחות כשהשאילתה שמאחוריהן מורכבת. למשל תצפית שמסכמת לכל אדם כמה תאונות מעורבות מכוניותיו (כולל מי שיש לו 0) — בעזרת צירוף חיצוני שלמדנו בשיעור 6:

create view pers_acc as
  select person.id, person.fname, person.lname,
         count(accident.accident_id) as num_accidents
  from   person left outer join owns     on person.id = owns.id
                left outer join accident on owns.license = accident.license
  group by person.id, person.fname, person.lname;

select * from pers_acc
where  num_accidents = 0

אחרי שהתצפית קיימת, השאלה "מי לא היה באף תאונה?" מתקצרת ל‑select * from pers_acc where num_accidents = 0 (בנתונים: נועה וליאור).

5למה זה שימושי + drop view

  • שימוש חוזר: כותבים שאילתה מורכבת פעם אחת, ומשתמשים בה שוב ושוב בשם קצר.
  • פשטות: שאילתות מעל התצפית קצרות וקריאות.
  • הפשטה/אבטחה: אפשר לחשוף למשתמש תצפית מצומצמת במקום את הטבלאות הגולמיות.
  • למחיקת תצפית: drop view cars_per_brand.
⚠️ תצפית אינה מאחסנת נתונים — היא "חלון" אל הטבלאות. אם הנתונים בטבלאות משתנים, התצפית משקפת זאת אוטומטית בפנייה הבאה.

תרגול — כתבו ובדקו את עצמכם ✍️

נסו בעצמכם לפני "הצג פתרון". כתבו את ה‑create view ואחריו select מהתצפית, והריצו (▶) — תראו את התוצאה.

חימום ותרגול

תרגיל 1 — הגדר תצפית בשם recent_cars שמכילה את כל הרכבים משנת ייצור 2023 ואילך, והצג את כל השורות שבה.
create view recent_cars as
  select * from car where year >= 2023;

select * from recent_cars

התצפית היא בסך הכול שאילתה בשם. אפשר עכשיו לשלוף ממנה כמו מטבלה. בנתונים מוחזרים 8 רכבים (כל אלה משנת 2023 ואילך).

תרגיל 2 — הגדר תצפית בשם acc_per_garage עם garage_id ומספר התאונות שטיפל בהן כל מוסך, ואז הצג ממנה רק מוסכים שטיפלו ב‑3 תאונות או יותר.
create view acc_per_garage as
  select garage_id, count(*) as n
  from   accident
  group by garage_id;

select * from acc_per_garage
where  n >= 3

קודם בונים את התצפית (ספירה לכל מוסך), ואז מסננים מעליה. שימו לב — הסינון n >= 3 נעשה ב‑where רגיל על התצפית, לא ב‑having, כי מבחינתנו acc_per_garage היא כבר "טבלה". בנתונים מוחזרים 6 מוסכים.

שאלות ממבחנים אמיתיים 🎓

🎓 תרגיל 3 (שאלה ממבחן 9.4) — הגדר תצפית בשם pers_acc שתציג לכל אדם את מספר התאונות שבהן היו מעורבות מכוניות בבעלותו (גם אנשים שאף אחת ממכוניותיהם לא הייתה מעורבת בתאונה צריכים להופיע, עם 0).

create view pers_acc as

         count(accident.accident_id) as num_accidents
  from   person left outer join owns     on person.id = owns.id
                left outer join accident on owns.license = accident.license

זו בדיוק תבנית הצירוף החיצוני + ספירת ה‑0 משיעור 6, רק עטופה ב‑create view. הספירה count(accident.accident_id) (ולא count(*)) נותנת 0 למי שאין לו תאונות. בנתונים יש 16 שורות; נועה וליאור מקבלים 0.

סיכום

תצפית = create view <שם> as <select>, ואז שולפים ממנה כמו מטבלה. היא וירטואלית (לא מאחסנת נתונים), מצוינת לשימוש חוזר ולפישוט שאילתות מורכבות. כשבמבחן כתוב "הגדר תצפית" — עוטפים את ה‑select ב‑create view ... as.

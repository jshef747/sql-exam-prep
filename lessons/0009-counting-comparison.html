ספירה והשוואה · having count(...) >= N · "הכי הרבה" · השוואה בין תקופות

שתי משפחות שאלות שהבחינה אוהבת. (א) ספירה בקבוצה — "רכבים עם 2 בעלים או יותר", "אנשים שיש להם 3 מכוניות", "האדם עם הכי הרבה מכוניות". (ב) השוואה בין תקופות — "רכבים שב‑2024 היו בפחות תאונות מאשר ב‑2023", דרך תת‑שאילתות סקלריות מתואמות.

🎯 היעד: לדעת לספור בתוך קבוצה ולסנן לפי הספירה (group by … having count(…) >= N), למצוא את המקסימום של ספירה ("הכי הרבה") בלי >= all, ולהשוות שתי ספירות מתקופות שונות עבור אותה שורה.
↩️ עד כה: שליפה (1), צירוף (2), like (3), הקבצה (4), MAX/MIN (5), צירוף חיצוני ומיון (6), תת‑שאילתות (7), חלוקה/"לכל" (8). היום מחברים את שני הכלים החזקים מ‑4 ו‑7: group by … having count מצד אחד, ותת‑שאילתות סקלריות מתואמות מצד שני.

1ספירה בקבוצה: having count(…) >= N

בשיעור 4 ראינו ש‑group by מחלק את הטבלה לקבוצות, ש‑count(*) סופר כמה שורות יש בכל קבוצה, ושמסננים קבוצות לפי having. כאן זה הופך לתבנית מרכזית: "ישות עם לפחות N דברים" = קבץ לפי הישות, ספור, וסנן having count(...) >= N.

דוגמה — "רכבים שיש להם 2 בעלים או יותר". כל שורה ב‑owns היא צמד (בעלים, רכב); אם נקבץ לפי license ונספור, נקבל לכל רכב כמה בעלים יש לו:

select license, count(*) as num_owners
from   owns
group  by license
having count(*) >= 2

קוראים: "קבץ את שורות הבעלוּת לפי רכב, ספור כמה בעלים לכל רכב, והשאר רק רכבים עם ספירה ≥ 2".

כשהישות בטבלה אחרת — מקבצים אחרי צירוף

"אנשים שיש להם 3 מכוניות או יותר". הישות (אדם) בטבלת person, אבל הספירה (מכוניות) בטבלת owns — אז קודם מצרפים, ואז מקבצים לפי האדם:

select p.fname, p.lname, count(*) as num_cars
from   person p join owns o on p.id = o.id
group  by p.id, p.fname, p.lname
having count(*) >= 3

— אותו דבר עם פסיק ו‑where:

select p.fname, p.lname, count(*) as num_cars
from   person p, owns o
where  p.id = o.id
group  by p.id, p.fname, p.lname
having count(*) >= 3
💡 למה מקבצים לפי p.id ולא רק לפי השם? כי שני אנשים שונים עלולים לחלוק שם פרטי+משפחה. המפתח (p.id) מזהה את האדם באופן חד‑משמעי; את fname/lname מוסיפים ל‑group by רק כדי שמותר יהיה לבחור אותם ב‑select.

2"הכי הרבה" — המקסימום של ספירה

שאלה נפוצה: לא "מי שיש לו ≥ N", אלא "מי שיש לו הכי הרבה" — הישות עם הספירה המקסימלית. כאן יש מלכודת: max(count(*)) ישירות אינו חוקי (לא מקננים פונקציות צבירה). הפתרון הוא שני שלבים:

  1. ספירה לכל ישות → טבלת ביניים של ספירות.
  2. לקחת מתוכה את ה‑max, ולהשאיר את הישות/ויות שספירתן שווה לו.

דוגמה — "הרכב שהיה בהכי הרבה תאונות". טבלת הביניים היא (select count(*) from accident group by license); עליה לוקחים max:

select   a.license, count(*) as num_accidents
from     accident a
group by a.license
having   count(*) = (select max(c)
                     from   (select count(*) as c
                             from   accident
                             group  by license) x)

התת‑שאילתה הפנימית (select count(*) as c … group by license) היא טבלה נגזרת (subquery ב‑from) — שורה אחת לכל רכב עם הספירה שלו. ה‑select max(c) מעליה מחזיר את הספירה הגבוהה ביותר, וה‑having משאיר רק רכבים שספירתם שווה לה.

⚠️ למה לא >= all? בשיעור 5 ראינו ש"הכי גדול" = >= all. כאן זה רעיון נכון אבל מסוכן בכתיבה: having count(*) >= all (select count(*) … group by …) מערבב count עם all וצירופים — צורה שקשה ולא תמיד נתמכת. השתמשו בצורת ה‑max מטבלה נגזרת — היא שקולה, ברורה, ורצה תמיד.
💡 שימו לב: אם שתי ישויות חולקות את הספירה המקסימלית — שתיהן יחזרו. זה נכון: "הכי הרבה" כולל את כל מי שמחזיק בשיא. אם רוצים בדיוק אחד, מוסיפים שובר‑שוויון — אבל בבחינה מספיק להחזיר את כולם.

3השוואה בין תקופות — תת‑שאילתות סקלריות מתואמות

המשפחה השנייה: להשוות שתי ספירות עבור אותה שורה. למשל "רכבים שב‑2024 היו בפחות תאונות מאשר ב‑2023". לכל רכב צריך לחשב שני מספרים — מספר התאונות שלו ב‑2024 ומספר התאונות שלו ב‑2023 — ולהשוות ביניהם.

הכלי: תת‑שאילתה סקלרית מתואמת — תת‑שאילתה שמחזירה ערך בודד (מספר) ומתייחסת לשורה החיצונית (דרך license = c.license). שמים אחת לכל תקופה, ומשווים:

select c.license, c.model, c.brand, c.year
from   car c
where  (select count(*) from accident
        where  license = c.license and [date] like '2024%')
       <
       (select count(*) from accident
        where  license = c.license and [date] like '2023%')

קוראים: "לכל רכב c, ספור כמה תאונות שלו ב‑2024 (תת‑שאילתה ראשונה), ספור כמה ב‑2023 (תת‑שאילתה שנייה), והשאר את הרכב רק אם המספר של 2024 קטן מזה של 2023". ה‑like '2024%' תופס את כל התאריכים שמתחילים ב‑2024 (השדה [date] בפורמט 'YYYY-MM-DD').

💡 למה תת‑שאילתה סקלרית ולא group by? כי group by ייתן שורה אחת לכל (רכב, שנה) — ואז קשה להעמיד "2024 מול 2023 של אותו רכב" באותה שורה. תת‑שאילתות סקלריות מתואמות מחשבות את שני המספרים בתוך אותה שורת רכב, וכך ההשוואה ישירה. זה בדיוק הכוח של ה‑correlation משיעור 7.

count(...) per group· having count = (select max ...)· scalar(2024) < scalar(2023)

תרגול — כתבו ובדקו את עצמכם ✍️

נסו בעצמכם לפני "הצג פתרון". הרמה עולה: חימום → תרגול → שאלות ממבחנים אמיתיים. המטרה שתדעו לפתור כל שאלה שבמבחנים.

חימום ותרגול

תרגיל 1 (ספירה ≥ N) — הצג את ה‑license ומספר הבעלים של כל רכב שיש לו 2 בעלים או יותר.
select license, count(*) as num_owners
from   owns
group  by license
having count(*) >= 2

"2 בעלים או יותר" ⟵ group by license having count(*) >= 2. בדוגמה מוחזרים 3 רכבים.

תרגיל 2 (ספירה ≥ N עם צירוף) — הצג שם פרטי, משפחה ומספר המכוניות של כל אדם שבבעלותו 3 מכוניות או יותר.
select p.fname, p.lname, count(*) as num_cars
from   person p join owns o on p.id = o.id
group  by p.id, p.fname, p.lname
having count(*) >= 3

— אותו דבר עם פסיק ו‑where:

select p.fname, p.lname, count(*) as num_cars
from   person p, owns o
where  p.id = o.id
group  by p.id, p.fname, p.lname
having count(*) >= 3

קודם מצרפים אדם↔בעלוּת, מקבצים לפי p.id, וסופרים. בדוגמה מוחזר אדם אחד (David Levi, 3 מכוניות).

תרגיל 3 (ספירה ≥ N על תאונות) — הצג license, model ומספר התאונות של כל רכב שהיה מעורב ב2 תאונות או יותר.
select c.license, c.model, count(*) as num_accidents
from   car c join accident a on c.license = a.license
group  by c.license, c.model
having count(*) >= 2

צירוף car↔accident, קיבוץ לפי הרכב, וספירת התאונות. בדוגמה מוחזרים 6 רכבים.

תרגיל 4 ("הכי הרבה") — הצג את ה‑license של הרכב/ים שהיו בהכי הרבה תאונות, ואת מספר התאונות שלו/הם.
select   a.license, count(*) as num_accidents
from     accident a
group by a.license
having   count(*) = (select max(c)
                     from   (select count(*) as c
                             from   accident
                             group  by license) x)

"הכי הרבה" = הספירה שווה ל‑max של כל הספירות (טבלה נגזרת). אם כמה רכבים חולקים את השיא — כולם יחזרו (בדוגמה: 3 רכבים עם 3 תאונות כל אחד).

תרגיל 5 ("הכי הרבה" עם צירוף) — הצג שם פרטי ומשפחה של האדם/ים שבבעלותו הכי הרבה מכוניות, ומספר המכוניות.
select   p.fname, p.lname, count(*) as num_cars
from     person p join owns o on p.id = o.id
group by p.id, p.fname, p.lname
having   count(*) = (select max(c)
                     from   (select count(*) as c
                             from   owns
                             group  by id) x)

אותו דפוס כמו תרגיל 4, אבל הישות (אדם) דורשת צירוף, והטבלה הנגזרת סופרת לכל id כמה רכבים. בדוגמה: David Levi עם 3 מכוניות.

תרגיל 6 (השוואה בין תקופות — הכיוון ההפוך) — הצג license, model, brand של רכבים שב‑2024 היו ביותר תאונות מאשר ב‑2023.
select c.license, c.model, c.brand
from   car c
where  (select count(*) from accident
        where  license = c.license and [date] like '2024%')
       >
       (select count(*) from accident
        where  license = c.license and [date] like '2023%')

שתי תת‑שאילתות סקלריות מתואמות (license = c.license), אחת לכל שנה, והשוואה >. בדוגמה מוחזרים 9 רכבים.

תרגיל 7 (ספרייה, "הכי הרבה") — הצג את שם השואל/ים (name) שהשאיל הכי הרבה ספרים, ואת מספר ההשאלות.

טבלאות: Borrowers(borrower_id, name, …) · Loans(loan_id, book_isbn, borrower_id, …)

select   br.name, count(*) as num_loans
from     Borrowers br join Loans l on br.borrower_id = l.borrower_id
group by br.borrower_id, br.name
having   count(*) = (select max(c)
                     from   (select count(*) as c
                             from   Loans
                             group  by borrower_id) x)

אותו דפוס "הכי הרבה" בסכמת הספרייה. בדוגמה יש שוויון בשיא, ולכן מוחזרים 2 שואלים (4 השאלות כל אחד).

שאלות ממבחנים אמיתיים 🎓

עכשיו ברמת הבחינה. היעזרו ב‑מילון הסכמות אם צריך.

🎓 תרגיל 8 (שאלה ממבחן 9.2 — "הכי הרבה") — מצא האדם שיש לו הכי הרבה מכוניות שהשתתפו בתאונה. הצג שם פרטי, משפחה, ומספר המכוניות.

📄 מקור: מבחן 131112 · 2023 · סמסטר 2 · מועד 1 (קובץ …_16208) · שאלה 9.2 · סכמה ב' — חברת ביטוח רכב

רמז: סופרים לכל אדם כמה מהמכוניות שבבעלותו היו בתאונה — לא כמה תאונות. כדי לספור מכוניות (ולא תאונות), מצרפים את owns לקבוצת ה‑license הייחודיים שמופיעים ב‑accident.

select   p.fname, p.lname, count(*) as num_cars
from     person p join owns o on p.id = o.id
         join (select distinct license from accident) al
              on o.license = al.license
group by p.id, p.fname, p.lname
having   count(*) = (select max(cc)
                     from   (select count(*) as cc
                             from   owns o2
                                    join (select distinct license from accident) al2
                                         on o2.license = al2.license
                             group  by o2.id) x)

הצירוף ל‑(select distinct license from accident) משאיר רק מכוניות שהיו בלפחות תאונה אחת — ושומר שורה אחת לכל מכונית כזו (לא לכל תאונה!), כדי שהספירה תהיה ספירת מכוניות. אחר כך זו אותה תבנית "הכי הרבה" כמו תרגיל 5. בדוגמה: David Levi עם 3 מכוניות שהיו בתאונה.

מלכודת: אילו היינו מצרפים ישירות owns join accident, מכונית שהייתה ב‑3 תאונות הייתה נספרת 3 פעמים — והתשובה הייתה ספירת תאונות, לא מכוניות. ה‑distinct license (או count(distinct o.license)) הוא מה שעושה את ההבדל.

🎓 תרגיל 9 (שאלה ממבחן 6.4 — ספירה ≥ 2 עם סינון) — הצג את רשומות התאונה של מכוניות שיש להן יותר מבעלים אחד (שני בעלים או יותר) שגרים בעיר חיפה (Haifa). הצג את כל פרטי רשומת התאונה. (כתבו [date] במנוע ההרצה.)

📄 מקור: מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480) · שאלה 6.4 · סכמה ב' — חברת ביטוח רכב

select a.accident_id, a.license, a.[date], a.driver_id, a.damage, a.garage_id
from   accident a
where  a.license in
       (select   o.license
        from     owns o join person p on o.id = p.id
        group by o.license
        having   count(*) >= 2
             and sum(case when p.city = 'Haifa' then 1 else 0 end) >= 1)

הליבה היא תת‑שאילתה שמחזירה את ה‑license של מכוניות "כשרות": מקבצים בעלוּת+אדם לפי רכב, דורשים count(*) >= 2 (שני בעלים או יותר), ובאותו having דורשים שלפחות אחד מהם גר בחיפה (sum(case … >= 1). אחר כך החיצונית שולפת את רשומות התאונה של כל המכוניות האלו עם in. בדוגמה מוחזרות 5 רשומות תאונה (מכוניות 77‑777‑77 ו‑12‑121‑12).

— הסבר ה‑sum(case …): זו "ספירה מותנית" — count רק של הבעלים מחיפה. >= 1 = "לפחות בעלים אחד מחיפה". זהו דפוס שימושי לשילוב תנאי בתוך הקבצה.

🎓 תרגיל 10 (שאלה ממבחן 6.5 — השוואה בין תקופות) — מצא מכוניות (פרטים מלאים של המכונית) שבשנת 2024 השתתפו בפחות תאונות מאשר בשנת 2023.

📄 מקור: מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480) · שאלה 6.5 · סכמה ב' — חברת ביטוח רכב

select c.license, c.model, c.brand, c.year
from   car c
where  (select count(*) from accident
        where  license = c.license and [date] like '2024%')
       <
       (select count(*) from accident
        where  license = c.license and [date] like '2023%')

"פרטים מלאים של המכונית" ⟵ כל עמודות car. שתי תת‑שאילתות סקלריות מתואמות (license = c.license), אחת סופרת תאונות 2024 והשנייה 2023, וההשוואה < משאירה רכבים שהמספר של 2024 קטן מזה של 2023. בדוגמה מוחזרות 5 מכוניות (כולל 55‑555‑55 ו‑77‑777‑77).

שימו לב: רכב שלא היה באף תאונה באף שנה מקבל 0 < 0 = שקר ולכן לא חוזר. רכב עם תאונות רק ב‑2023 ולא ב‑2024 מקבל 0 < (חיובי) = אמת וכן חוזר — זה בדיוק "פחות ב‑2024".

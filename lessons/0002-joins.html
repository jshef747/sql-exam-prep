שיעור 2 · צירוף טבלאות

joins — לחבר מידע מכמה טבלאות

הצעד הגדול הבא. כמעט כל שאלת SQL בבחינה דורשת לאסוף מידע שמפוזר על פני שתי טבלאות או יותר. אחרי השיעור הזה תדעו לחבר אותן נכון.

🎯 היעד: לכתוב שאילתה ששולפת מידע מכמה טבלאות יחד — למשל "מה שם הבעלים של רכב?", כששם הבעלים נמצא בטבלה person אבל הקשר לרכב נמצא בטבלה owns.
↩️ מה היה בשיעור 1: select (אילו עמודות), from (מאיזו טבלה), where (אילו שורות), ו‑distinct. עכשיו ה‑from יקבל יותר מטבלה אחת.

הסכמה שנעבוד עליה

שוב סכמת חברת ביטוח הרכב. שימו לב לעמודות המשותפות — הן ה"דבק" שמחבר בין הטבלאות (מפתח ראשי בקו תחתון):

טבלהעמודותמתחבר ל…
personid, fname, lname, city, …—
ownsid, licenseid→person · license→car
carlicense, model, brand, year—
accidentaccident_id, license, date, damage, garage_idlicense→car · garage_id→garage
garagegarage_id, garage_name, city, …—

סכמה מלאה ב‑מילון המונחים והסכמות ←

1הבעיה: המידע מפוזר

נניח שאנחנו רוצים את שמות הבעלים של כל הרכבים. השמות (fname, lname) יושבים ב‑person, אבל מי מחזיק איזה רכב יושב ב‑owns. אף טבלה לבדה לא מספיקה — צריך לחבר אותן.

person— id —owns— license —car

החיבור נעשה דרך העמודה המשותפת: person.id שווה ל‑owns.id.

2הצירוף הבסיסי: שתי טבלאות ב‑from

פשוט שמים את שתי הטבלאות ב‑from (מופרדות בפסיק) ומוסיפים את תנאי הצירוף ב‑where:

select distinct person.fname, person.lname
from   person, owns
where  person.id = owns.id

תנאי הצירוף person.id = owns.id הוא הלב של העניין. הוא מתאים כל אדם לרכבים שלו.

⚠️ הטעות הכי מסוכנת בבחינה — לשכוח את תנאי הצירוף. בלי ה‑where, שתי טבלאות ב‑from נותנות מכפלה קרטזית (×): כל שורה ב‑person משולבת עם כל שורה ב‑owns — אלפי שילובים חסרי משמעות. תמיד ודאו שכל טבלה נוספת מחוברת בתנאי.

הקשר לאלגברת היחסים

תנאי הצירוף על העמודה המשותפת הופך את המכפלה הקרטזית ל‑צירוף טבעי (מצגת פרק 3, שקופית 7):

 σperson.id = owns.id( person × owns )  ≈  person ⋈ owns 

(הצירוף הטבעי ⋈ ממזג את עמודת id המשותפת לעמודה אחת; הצירוף עם where משאיר את שתיהן.)

ואז ה‑select מטיל (π) את העמודות שביקשנו — אחרי הצירוף:

 πfname,lname( σperson.id = owns.id( person × owns ) ) 

3שלוש דרכים לכתוב את אותו צירוף

המצגת (פרק 3, שקופית 8) מציגה שלוש כתיבות שקולות. כולן נכונות — בבחינה בחרו את מה שנוח לכם, אבל דעו לזהות את כולן:

א · פסיק + where (הקלאסי)

select distinct person.fname, person.lname
from   person, owns
where  person.id = owns.id

ב · join ... on

select distinct person.fname, person.lname
from   person join owns on (person.id = owns.id)

ג · צירוף טבעי / using

select distinct fname, lname
from   person natural join owns       -- מחבר אוטומטית לפי כל עמודה משותפת (כאן: id)
💡 צירוף טבעי מחבר לפי כל העמודות בעלות אותו שם. כאן ל‑person ול‑owns משותפת רק id, אז זה עובד מצוין. היזהרו ממנו כששתי טבלאות חולקות עמודה במקרה (למשל city מופיע גם ב‑person וגם ב‑garage).

4שלוש טבלאות בשרשרת

כדי לקבל את שם הבעלים + יצרן הרכב צריך לעבור דרך שלוש טבלאות: person → owns → car. מוסיפים טבלה, ולכל קישור — תנאי צירוף משלו:

select distinct person.fname, person.lname, car.brand
from   person, owns, car
where  person.id = owns.id
  and  owns.license = car.license

שימו לב: שני תנאי צירוף (אחד לכל "חוליה" בשרשרת), מחוברים ב‑and. כלל אצבע: ל‑N טבלאות צריך לפחות N−1 תנאי צירוף.

5הבהרת עמודות וכינויים (aliases)

כש‑עמודה באותו שם קיימת ביותר מטבלה אחת (כמו id או license), חובה לציין מאיזו טבלה: person.id, לא סתם id (אחרת השאילתה דו‑משמעית).

אפשר לתת לטבלה כינוי קצר כדי לחסוך כתיבה (מצגת פרק 3, שקופית 14):

select distinct p.fname, p.lname, c.brand
from   person p, owns o, car c
where  p.id = o.id
  and  o.license = c.license

הכינויים (p, o, c) הם גם הדרך היחידה לחבר טבלה לעצמה — טריק שנפגוש בשיעור מתקדם יותר.

🔁 למה distinct? צירוף עלול לייצר שורות כפולות (אדם עם שני רכבים יופיע פעמיים). אם רוצים כל שם פעם אחת — הוסיפו distinct.

תרגול — כתבו ובדקו את עצמכם ✍️

השתמשו בעורך עם ההשלמה האוטומטית. נסו בעצמכם לפני "הצג פתרון". טיפ: התחילו תמיד מהשאלה "אילו טבלאות אני צריך?" ואז "איך הן מתחברות?".

תרגיל 1 — הצג את השמות המלאים של כל בעלי הרכבים (כל בעלים פעם אחת).
select distinct fname, lname
from   person join owns on person.id = owns.id

— אותו דבר עם פסיק ו‑where:

select distinct person.fname, person.lname
from   person, owns
where  person.id = owns.id

— או צירוף טבעי (עמודה משותפת: id):

select distinct fname, lname
from   person natural join owns

שתי טבלאות: השמות ב‑person, הקשר לבעלוּת ב‑owns. תנאי הצירוף על id. distinct כי אדם עם כמה רכבים יופיע אחרת כמה פעמים. שלוש הצורות שקולות.

תרגיל 2 — לכל תאונה, הצג את מספר התאונה (accident_id) ואת יצרן הרכב (brand) שהיה מעורב בה.
select accident.accident_id, car.brand
from   accident join car on accident.license = car.license

— אותו דבר עם פסיק ו‑where:

select accident.accident_id, car.brand
from   accident, car
where  accident.license = car.license

— או צירוף טבעי (עמודה משותפת: license):

select accident_id, brand
from   accident natural join car

ל‑accident יש license של הרכב המעורב; מחברים ל‑car כדי לקבל את ה‑brand. כאן בדרך כלל לא צריך distinct — כל שורה היא רכב מסוים בתאונה מסוימת.

תרגיל 3 — הצג את שמותיהם של האנשים שמחזיקים רכב מתוצרת 'Tesla'.
select distinct fname, lname
from   person join owns on person.id = owns.id
              join car  on owns.license = car.license
where  car.brand = 'Tesla'

— אותו דבר עם פסיק ו‑where:

select distinct person.fname, person.lname
from   person, owns, car
where  person.id = owns.id
  and  owns.license = car.license
  and  car.brand = 'Tesla'

— או צירוף טבעי (שרשרת):

select distinct fname, lname
from   person natural join owns natural join car
where  brand = 'Tesla'

שרשרת של שלוש טבלאות (שני תנאי צירוף), ועוד תנאי סינון רגיל על brand. שילוב מושלם של שיעור 1 (סינון) ושיעור 2 (צירוף). שימו לב שתנאי הסינון (brand) נשאר ב‑where גם בצורת ה‑join.

תרגיל 4 — הצג את שם המוסך (garage_name) שטיפל בתאונה שמספרה 100.
select distinct garage.garage_name
from   accident join garage on accident.garage_id = garage.garage_id
where  accident.accident_id = 100

— אותו דבר עם פסיק ו‑where:

select distinct garage.garage_name
from   accident, garage
where  accident.garage_id = garage.garage_id
  and  accident.accident_id = 100

— או צירוף טבעי (עמודה משותפת: garage_id):

select distinct garage_name
from   accident natural join garage
where  accident_id = 100

תנאי הצירוף על garage_id, ועוד תנאי סינון על מספר התאונה. (מספר תאונה הוא מספר — בלי גרשיים.)

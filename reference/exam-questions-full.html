שאלות המבחן — הגרסה המלאה + ניתוח פערים

הקובץ מרכז את שאלות ה‑SQL מהמבחנים שבשיעורים השתמשנו בהן באופן חלקי (קיצרנו כדי להתאים לנושא השיעור), ושאלות שלא כוסו כלל — עם הניסוח המלא, מקור, סכמה, מה הושמט, והפתרון/ההבנה המלאה.

איך לקרוא את הקובץ

בדקתי את כל עשרת השיעורים. שאלה שמופיעה במלואה כבר באחד השיעורים — לא חוזרת כאן (ראו את טבלת "כוסו במלואן" בסוף). כאן נמצאות רק:

  • חלקי שאלות שבשיעור הוצג רק חלק מהן (התאמה לנושא) — כאן הגרסה המלאה.
  • לא כוסה שאלות שלא צוטטו באף שיעור.

סכמות: ב' = חברת ביטוח רכב (person, owns, car, accident, garage) · ג' = ספרייה. ראו מילון הסכמות.

חלקי שאלה 6.2 — התאמת מחרוזות + צירוף שלוש טבלאות

📄 מקור: מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480) · שאלה 6.2 · סכמה ב'

הניסוח המלא: "מצא את התאונות, שבהן היו מעורבות המכוניות בבעלותם של לקוחות ששם המשפחה שלהם מכיל את הרצף tov (למשל שם משפחה shem tov levi או simantov וכו'). יש להציג: מספר תאונה, מס' רישיון מכונית, ותאריך, ת"ז ושם מלא (פרטי ומשפחה) של הבעלים. ניתן להניח שכל השמות הם באותיות קטנות (lowercase)."
✔️ כוסה בשיעור 3 (התאמת מחרוזות).
✂️ מה הושמט בשיעור: עמודת ה‑ת"ז של הבעלים (person.id) בתצוגה, וההנחה המפורשת על lowercase. בשיעור הוצגו רק accident_id, license, date ושם הבעלים.
select accident.accident_id, accident.license, accident.date,
       person.id, person.fname, person.lname
from   accident join owns   on accident.license = owns.license
               join person on owns.id = person.id
where  person.lname like '%tov%'

— אותו דבר עם פסיק ו‑where:

select accident.accident_id, accident.license, accident.date,
       person.id, person.fname, person.lname
from   accident, owns, person
where  accident.license = owns.license
  and  owns.id = person.id
  and  person.lname like '%tov%'

ההבדל מהשיעור: הוספת person.id (ת"ז) לרשימת ה‑select. ההתאמה like רגישה לאותיות גדולות/קטנות, ולכן ההנחה "הכול lowercase" מצדיקה לכתוב '%tov%' באותיות קטנות.

חלקי שאלה 9.4 — הגדרת תצפית (create view)

📄 מקור: מבחן 131112 · 2023 · סמסטר 2 · מועד 1 (קובץ …_16208) · שאלה 9.4 · סכמה ב'

הניסוח המלא: "הגדר תצפית בשם pers_acc שתציג לכל אדם מספר התאונות בהן היו מעורבות מכוניות בבעלותו. (גם כאלה שאף אחת ממכוניותיו לא היתה מעורבת בהתנגשות צריכים להופיע בתצפית.)"
✔️ ה‑select הפנימי כוסה בשיעור 6 (צירוף חיצוני + ספירת אפס); המושג create view נלמד בשיעור 10 (DDL).
✂️ מה הושמט בשיעור 6: עטיפת ה‑create view pers_acc as (...) סביב ה‑select. השאלה מבקשת תצפית, לא רק שאילתה.
create view pers_acc as
  select person.id, person.fname, person.lname,
         count(accident.accident_id) as num_accidents
  from   person left outer join owns     on person.id = owns.id
                left outer join accident on owns.license = accident.license
  group by person.id, person.fname, person.lname

הליבה זהה לשיעור 6: שני צירופים חיצוניים שומרים כל אדם, ו‑count(accident.accident_id) נותן 0 (ולא 1) למי שאין לו תאונות. ההשלמה היחידה היא העטיפה create view pers_acc as — וכך אפשר אחר כך לכתוב select * from pers_acc.

חלקי שאלה 9.1 — במקור באלגברת יחסים

📄 מקור: מבחן 131112 · 2023 · סמסטר 2 · מועד 1 (קובץ …_16208) · שאלה 9.1 · סכמה ב'

הניסוח המלא: "אלגברת יחסים: לכל תאונה שבה הנהג (driver) אינו אחד מהבעלים של הרכב, הצג את פרטי התאונה (license, date, driver_id, damage)."
✔️ כוסה בשיעור 7 — אך כ‑SQL (not exists / not in).
✂️ מה הושמט: השאלה מבקשת אלגברת יחסים, והשיעור נתן SQL. להלן ה‑RA (ההבנה שלי), לצד ה‑SQL.

אלגברת יחסים — "כל התאונות" פחות "התאונות שבהן הנהג כן בעלים":

π[license,date,driver_id,damage] (accident)
  −
π[license,date,driver_id,damage] (
    σ[accident.driver_id = owns.id] ( accident ⋈[accident.license = owns.license] owns )
)

המשמעות: מצרפים תאונה לבעלים של אותו רכב (accident.license = owns.license), שומרים רק שורות שבהן הנהג הוא הבעלים (driver_id = owns.id), ומחסירים אותן מכלל התאונות.

SQL שקול (כפי שבשיעור 7):

select license, date, driver_id, damage
from   accident a
where  not exists (select *
                   from   owns
                   where  owns.license = a.license
                     and  owns.id = a.driver_id)

לא כוסה שאלה 6.1 — במקור באלגברת יחסים

📄 מקור: מבחן 131112 · 2025 · סמסטר 1 · מועד 2 (קובץ …_15480) · שאלה 6.1 · סכמה ב'

הניסוח המלא: "אלגברת יחסים: מצא את שמות בעלי המכוניות (שם פרטי, שם משפחה) שיש להם לפחות שתי מכוניות."
⚠️ התבנית "לפחות שתי מכוניות" (ספירה / צירוף‑עצמי) נלמדה בשיעור 9 כ‑SQL, אבל שאלה 6.1 עצמה (ניסוח RA, שמות בעלים) לא צוטטה.

אלגברת יחסים — בעל עם ≥2 מכוניות = קיימות שתי שורות owns עם אותו id ורישיונות שונים (צירוף‑עצמי):

two_or_more ← π[O1.id] (
    σ[O1.id = O2.id  ∧  O1.license < O2.license] ( ρ[O1](owns) × ρ[O2](owns) )
)
answer ← π[fname,lname] ( person ⋈[person.id = two_or_more.id] two_or_more )

SQL שקול (תבנית שיעור 9):

select person.fname, person.lname
from   person join (select id
                    from   owns
                    group by id
                    having count(*) >= 2) t on person.id = t.id
